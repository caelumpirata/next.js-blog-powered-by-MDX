Deploy Production Ready Application on Ditgital Ocean Kubernetes with nginx and Let's encrypt.


## Prerequisites
* Create a Kuberntes Cluster from Digital ocean Control Plane
* Setup your domain in Networking panel in Digital Ocean Control Plane

## Installing Ingress-Nginx

Download <Link href="https://artifacthub.io/packages/helm/ingress-nginx/ingress-nginx">Ingress-Nginx</Link>

```bash
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo update
helm install ingress-nginx ingress-nginx/ingress-nginx
```

Deploy helm chart
```bash
helm install ingress-nginx ingress-nginx/ingress-nginx
```

## Adding Ingress

* Create ingress.yaml file.
* Add ingress host name in Digital Ocean Networking DNS.

eg. `grafana.example.com`

```bash
apiVersion: networking.k8s.io/v1 
kind: Ingress  
metadata:  
  name: demo-ingress 
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /$1
 
spec:
  rules:
    - host: grafana.example.com
      http:
        paths:
          - path: /(.*)
            pathType: Prefix
   
            backend:
   
              service:
   
                name: grafana
   
                port:
   
                  number: 80
    - host: nodered.example.com
   
      http:
   
        paths:
   
          - path: /(.*)
            pathType: Prefix
   
            backend:
   
              service:
   
                name: nodered-node-red
   
                port:
   
                  number: 1880

  ingressClassName: nginx
```

install it using
```bash
kubectl apply -f ingress.yaml
```


> Getting `HTTPS` using `Let's Encrypt`, we will use `Cert-Manger` here

## Installing Cert Manager

Download <Link href="https://artifacthub.io/packages/helm/cert-manager/cert-manager">Cert-Manager</Link>

Create a namespace cert-manager
```bash
kubectl create ns cert-manager
```
Installing crds using
```bash
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.11.0/cert-manager.crds.yaml
```
Adding Jetstack helm repo
```bash
helm repo add jetstack https://charts.jetstack.io
```
Installing the cert-manager helm chart
```bash
helm install cert-manager --namespace cert-manager --version v1.11.0 jetstack/cert-manager
```
Create ISSUER to sign certificates using private-key
```yaml
apiVersion: cert-manager.io/v1
kind: Issuer
metadata:
  name: letsencrypt-prod
spec:
  acme:
    email: [demo@email.com]
    server: https://acme-v02.api.letsencrypt.org/directory
    privateKeySecretRef:
      name: letsencrypt-nginx-private-key
    solvers:
      - http01:
          ingress:
            class: nginx
```
creating issuer
```
kubectl apply -f issuer.yaml
```
Edit your ingress.yaml deployment file
> Add cert-manager annotation in `metadata:` and `tls` in spec:
```yaml
apiVersion: networking.k8s.io/v1 
kind: Ingress  
metadata:  
  name: demo-ingress 
  annotations:
    cert-manager.io/issuer: letsencrypt-prod
    nginx.ingress.kubernetes.io/rewrite-target: /$1
 
spec:
  tls:
    - hosts:
      - nodered.exmpale.com
      - grafana.example.com
      secretName: letsencrypt-prod
  rules:
    - host: grafana.example.com
      http:
        paths:
          - path: /(.*)
            pathType: Prefix
   
            backend:
   
              service:
   
                name: grafana
   
                port:
   
                  number: 80
    - host: nodered.example.com
   
      http:
   
        paths:
   
          - path: /(.*)
            pathType: Prefix
   
            backend:
   
              service:
   
                name: nodered-node-red
   
                port:
   
                  number: 1880

  ingressClassName: nginx
```
re-deploy ingress.yaml
```
kubectl apply -f ingress.yaml
```
